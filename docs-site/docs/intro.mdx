# Mud Game documentation

This site exists to capture the parts of the project that are harder to infer from code alone: runtime architecture, local workflow, gameplay rules, and data-driven content conventions.

## What belongs here

- how to run the stack locally
- how the backend, Angular client, and world data fit together
- notes on gameplay systems that now have enough complexity to deserve stable documentation
- authoring rules for content that lives in JSON rather than code

## Where to start

- Read [Local Development](./getting-started/local-development.md) to boot the backend, frontend, and docs site.
- Read [Project Structure](./architecture/project-structure.md) for a repository map.
- Read [Movement, Resting, And Travel](./game-systems/movement-resting-and-travel.md) for one of the most actively evolving game systems.
- Read [World Data](./content/world-data.md) for the data-driven content layout.

## Documentation intent

The goal is not to duplicate every source file. The goal is to make the repo easier to change safely by capturing the rules, boundaries, and assumptions that are otherwise easy to lose in implementation alone.
